# Instruct

1. 注意制定plan.txt的时候，步骤需要详细一点。

3. 检查任何反馈循环，如类型检查和测试。
4. 将你的进度追加到 progress.txt 文件。
5. 提交该步骤的 git commit，所有涉及的中间文件，都保存到 git ，方便回溯查询。
6. 如果在完成步骤时，你注意到所有工作任务都已完成，创建一个空文件 done.txt 。
